# Day 2 operations for telco core CNF clusters



You can use the following Day 2 operations to manage telco core CNF clusters.

Updating a telco core CNF cluster:: Updating your cluster is a critical task that ensures that bugs and potential security vulnerabilities are patched.
For more information, see Updating a telco core CNF cluster.
Troubleshooting and maintaining telco core CNF clusters:: To maintain and troubleshoot a bare-metal environment where high-bandwidth network throughput is required, see Troubleshooting and maintaining Red Hat OpenShift Container Platform clusters.
Observability in telco core CNF clusters:: Red Hat OpenShift Container Platform generates a large amount of data, such as performance metrics and logs from the platform and the workloads running on it.
As an administrator, you can use tools to collect and analyze the available data.
For more information, see Observability in Red Hat OpenShift Container Platform.
Security:: You can enhance security for high-bandwidth network deployments in telco environments by following key security considerations.
For more information, see Security basics.
